# PE/PB 估值法

> 围绕"估算企业未来增长空间"的目的，构建一套**普通股 PE 主导 + 周期股 PB 主导**的双轨估值框架。

---

## 前置部分：估值前的数据准备

无论走 PE 还是 PB 路径，第一步都是**拉齐数据**。两条路径所需数据有约 60% 重叠，周期股需额外补充几类。

### 一、PE 估值所需数据（普通股）

#### 原始数据（akshare 拉取）

| 数据类别 | 具体字段 | 时间跨度 | 用途 |
|---|---|---|---|
| **股价** | 收盘价、总股本、总市值 | 近 5–10 年日频 | 计算历史 PE |
| **利润表** | 归母净利润、扣非净利润 | 近 5 年 + 当期 | EPS 输入 |
| **EPS** | 基本 EPS、稀释 EPS | 近 5 年 + 当期 | 静态 PE 输入 |
| **一致预期** | 未来 1–3 年净利润预测、增速预测 | 当前快照 | Forward PE + PEG |
| **行业归属** | 申万一级 / 二级行业代码 | 当前 | 锁定可比公司池 |
| **可比公司财务** | 同行业各家 PE、净利润、市值 | 当前快照 | 行业中位数 PE |

#### 衍生指标（本地计算）

| 衍生数据 | 计算方法 | 服务于 |
|---|---|---|
| 历史 PE 序列 | 每日股价 / 滚动 TTM EPS | 历史分位 PE |
| 历史 PE 分位数 | 当前 PE 在 5 年序列中的百分位 | 历史分位 PE |
| 历史中位数 PE | 5–10 年 PE 序列的中位数 | 历史分位法目标值 |
| 预测 EPS | 一致预期净利润 / 总股本 | PEG + Forward PE |
| 预测净利润增速 | 未来 3 年 CAGR 或明年增速 | PEG |
| 行业中位数 PE | 同行业 N 家公司当前 PE 的中位数 | 行业相对法 |

#### PE 最小数据集（MVP）

```python
required_data_pe = {
    "stock_code": "600519.SH",
    "current_price": float,
    "current_market_cap": float,
    "eps_ttm": float,
    "eps_forecast": float,
    "growth_forecast": float,
    "pe_history_5y": list[float],
    "industry_code": str,
    "peers_pe": list[float],
}
```

→ **9 个字段就能跑通 PE 三角验证**。

### 二、PB 估值额外所需数据（周期股）

PE 所需数据**全部保留**，再额外补充以下几类：

#### 原始数据（额外）

| 数据类别 | 具体字段 | 时间跨度 | 用途 |
|---|---|---|---|
| **资产负债表** | 归母净资产、每股净资产（BVPS） | 近 5–10 年 | 计算 PB |
| **EBITDA / EV** | EBITDA、企业价值 | 近 5 年 + 当期 | EV/EBITDA 估值 |
| **历史 EPS 序列** | 5–7 年 EPS | 近 5–7 年 | 穿越周期 EPS |
| **行业景气指标** | 商品价格、库存、产能利用率等 | 月频/日频 | 周期位置判断 |
| **实物量数据** | 产能、储量、销量 | 当期 | EV/产能、EV/储量 |

#### 衍生指标（额外）

| 衍生数据 | 计算方法 | 服务于 |
|---|---|---|
| 历史 PB 序列 | 每日股价 / BVPS | 历史分位 PB |
| 历史 PB 分位数 | 当前 PB 在 5 年序列中的百分位 | 周期位置判断 |
| 穿越周期 EPS | 5–7 年 EPS 算术平均 | 周期 PE 中枢 |
| 行业中位数 PB | 同行业 N 家公司当前 PB 的中位数 | 行业 PB 对比 |
| 景气度分位 | 商品价格 / 库存 / 价差 在历史中的分位 | 区间上下沿选择 |

#### PB 最小数据集（MVP）

```python
required_data_pb = {
    **required_data_pe,                  # 继承 PE 所需的 9 个字段
    "net_assets": float,                 # 归母净资产
    "bvps": float,                       # 每股净资产
    "pb_history_5y": list[float],        # 5 年 PB 序列
    "eps_history_7y": list[float],       # 7 年历史 EPS
    "industry_pb_median": float,         # 行业 PB 中位数
    "industry_boom_indicator": float,    # 景气指标当前值
    "industry_boom_percentile": float,   # 景气指标历史分位
    "ebitda": float,                     # 当期 EBITDA
    "ev": float,                         # 企业价值
}
```

→ **额外 9 个字段，总计 18 个字段跑通周期股估值**。

#### 关键差异提醒（vs 商业终端）

| 维度 | akshare 现状 | 应对策略 |
|---|---|---|
| **接口稳定性** | 依赖东财/新浪/同花顺等公开页面，源站改版即失效 | 抓取层加 try/except + 重试，关键数据多源交叉 |
| **调用频率** | 无显式配额，但高频会被封 IP | 每次调用间加 `time.sleep(0.5–1.0)` |
| **一致预期覆盖度** | 大盘股可用，**小盘股 / ST 股可能缺失** | 缺失时回退到自建预测（历史增速外推） |
| **EBITDA** | **不直接提供**，需本地计算 | EBITDA ≈ 营业利润 + 折旧 + 摊销 + 利息支出（从利润表/现金流量表拼装） |
| **字段名** | 多为中文（如"归属于母公司股东的净利润"） | 在 `statements.py` 里建一份中英文字段映射字典 |
| **历史深度** | PE/PB 序列最早回溯到 2005 年左右 | 周期股建议尽量取满 10 年 |

### 四、数据质量检查清单

抓数据后必须验证：

**通用检查**
- [ ] EPS 是否剔除一次性损益（优先用扣非净利润）
- [ ] PE/PB 序列是否剔除异常期（EPS ≤ 0 时 PE 无意义；净资产为负时 PB 无意义）
- [ ] 一致预期覆盖的分析师数量（< 3 家可信度低）
- [ ] 可比公司池是否合理（剔除业务差异过大的公司）
- [ ] 行业中位数是否剔除极端值（去掉最高/最低 10%）

**周期股专属检查**
- [ ] 历史 EPS 序列是否覆盖完整周期（至少 5 年，最好 7–10 年）
- [ ] 行业景气指标频率是否匹配（月频 vs 季频财报）
- [ ] 是否有送转股 / 增发导致 BVPS 不可比（需复权处理）

**akshare 专属检查**
- [ ] 抓取函数是否加了 `time.sleep()` 防封 IP
- [ ] 中文字段名是否做了到英文 key 的映射（避免列名空格/全角差异引发 KeyError）
- [ ] 一致预期缺失时是否有 fallback（如：以历史 3 年 CAGR 外推）
- [ ] EBITDA 是否本地拼装完成（akshare 不直接提供）
- [ ] akshare 版本是否最新（`pip show akshare`）

## 第一部分：普通股 PE 估值框架

### 一、PE 组合判断矩阵

**作用**：六维度快速扫描，决定"这家公司估值层面值不值得深入研究"。

| 指标 | 判断逻辑 | 正面信号触发标准 |
|---|---|---|
| **TTM PE** | 当前估值绝对水平 | 低于行业中位数 |
| **Forward PE** | 预期业绩兑现后的估值 | 明显低于 TTM PE（暗示 EPS 将增长） |
| **行业对比** | 行业内相对位置 | 估值低于行业 + 增速高于行业 |
| **历史分位** | 自身历史中的位置 | < 30% 偏低 / 30–70% 合理 / > 70% 偏高 |
| **PEG** | 估值与增速匹配度 | < 1 低估 / = 1 合理 / > 1 高估 |
| **目标 PE 测算** | 中性情景三年潜在空间 | 空间 > 50% 视为有吸引力 |

#### 示例

| 指标 | 实际值 | 判断 |
|---|---|---|
| TTM PE | 20x（行业中位数 22x） | 正面 |
| Forward PE | 14.3x（< TTM PE） | 正面 |
| 行业对比 | 估值低 + 增速高 | 正面 |
| 历史分位 | 30%（偏低） | 正面 |
| PEG | 0.8（< 1） | 正面 |
| 目标 PE 测算 | 三年空间 90% | 正面 |

→ **6/6 全正面 → 强信号候选**。

#### 汇总规则

| 正面数量 | 结论 | 下一步 |
|---|---|---|
| ≥ 5 | 强信号 | 进入三角验证深度测算 |
| 3–4 | 中性 | 配合验证排雷模块再判断 |
| ≤ 2 | 估值无吸引力 | 跳过，除非有强催化剂 |

> ⚠️ 全正面也要警惕"价值陷阱"——便宜可能是因为基本面有雷，必须交叉验证排雷模块。

---

### 二、三角验证

**作用**：对矩阵筛出的标的，算出具体估值区间与增长空间。

#### 三种方法各算一个目标市值

| 方法 | 公式 | 目标市值 |
|---|---|---|
| **PEG 法** | (PEG=1 × 预测增速) × 预测 EPS × 总股本 | V₁ |
| **历史分位法** | 历史中位数 PE × 预测 EPS × 总股本 | V₂ |
| **行业相对法** | 行业中位数 PE × 预测 EPS × 总股本 | V₃ |

#### 取交集得到估值锚

```
                       V₁ ── PEG 法
                        \
当前市值 ─── 偏离度 ───  估值锚区间 [V_low, V_high]  ← V₂ ── 历史分位法
                        /
                       V₃ ── 行业相对法

增长空间 = (V_中位 − 当前市值) / 当前市值
```

#### 三档情景分析

| 情景 | 假设 | 输出 |
|---|---|---|
| 悲观 | 预测 EPS × 0.8，取 V_low | 下行风险 |
| 中性 | 预测 EPS × 1.0，取 V_中位 | 合理估值 |
| 乐观 | 预测 EPS × 1.2，取 V_high | 上行空间 |

---

### 三、矩阵与三角验证的关系

```
┌───────────────────────────────────────────────────┐
│  矩阵（定性筛选）─────→  判断"是否值得估值"        │
│         │                                          │
│         ▼ ≥ 5 正面信号                             │
│  三角验证（定量测算）──→  算出"具体估值区间"      │
│         │                                          │
│         ▼                                          │
│  情景分析（区间输出）──→  悲观/中性/乐观三档      │
└───────────────────────────────────────────────────┘
```

**一句话**：矩阵决定"看不看"，三角验证决定"值多少"，情景分析决定"风险敞口多大"。

## 第二部分：强周期股 PB 估值调整

### 一、为什么传统 PE 失效

| 周期阶段 | 业绩 | 股价 | TTM PE | 市场误判 |
|---|---|---|---|---|
| **底部** | 巨亏或微利 | 已先行下跌后企稳 | 极高 / 负数 / 无意义 | "太贵了，不买" → 错过底部 |
| **复苏** | 业绩反转向上 | 跟随上涨 | 快速下降 | "估值合理" → 开始买入 |
| **顶部** | 业绩创新高 | 涨到最高 | 看起来最便宜（5–8x） | "好便宜，加仓" → 接盘 |
| **衰退** | 业绩断崖 | 快速下跌 | 急速抬升 | "贵了，卖出" → 卖在半山腰 |

**经典口诀**：**周期股 PE 低时不买，PE 高时不卖**。

---

### 二、强周期股识别

| 判断维度 | 强周期特征 |
|---|---|
| 行业类型 | 有色、煤炭、钢铁、化工、航运、券商、地产、养殖 |
| 营收波动 | 5 年内出现过 ≥ 30% 同比下滑 |
| 净利润波动 | 历史曾出现亏损或利润下滑 > 50% |
| 毛利率波动 | 高低点差距 > 15 个百分点 |
| 与商品/景气指数相关性 | 与铜价、油价、运价、利率等高度相关 |

→ **满足 ≥ 3 条 → 按强周期股处理**。

---

### 三、五项核心估值调整

#### 1. 用 PB 替代 PE 作为主估值锚

| 对比 | PE 估值 | PB 估值 |
|---|---|---|
| 分母 | EPS（剧烈波动） | 净资产（相对稳定） |
| 周期底部 | 失真 | 仍可用 |
| 推荐用法 | 仅作参考 | **强周期股首选** |

判读：
- PB < 1：跌破净资产，常见于周期底部
- PB 历史分位 < 20% + 行业景气未恢复 → 左侧布局信号
- PB 历史分位 > 80% + 业绩创新高 → 周期顶部警示

#### 2. 用"穿越周期 EPS"替代当期 EPS

| 替代指标 | 计算方法 | 优势 |
|---|---|---|
| **5–7 年平均 EPS** | 取过去一整个周期的算术平均 | 平滑波动 |
| **中周期 EPS** | 取历史高点和低点的中位数 | 反映正常盈利水平 |
| **Shiller PE（CAPE）** | 当前股价 / 10 年通胀调整后平均 EPS | 学术标准做法 |

→ **PE_normalized = 当前股价 / 平均 EPS** 才是有意义的估值参考。

#### 3. 历史分位 PE 改为历史分位 PB

| 历史分位 PB | 周期位置判断 | 操作含义 |
|---|---|---|
| < 20% | 接近周期底部 | 左侧关注 |
| 20–50% | 复苏早期 | 配置窗口 |
| 50–80% | 景气中后期 | 持有 |
| > 80% | 接近顶部 | 减仓 |

#### 4. 加入景气度指标作为外部锚

| 行业 | 景气指标 |
|---|---|
| 有色（铜、铝） | LME 价格、库存、产能利用率 |
| 煤炭 | 动力煤/焦煤价格、库存 |
| 化工 | 价差（产品价格 - 原材料价格） |
| 航运 | BDI、CCFI 指数 |
| 券商 | 日均成交额、两融余额 |
| 养殖 | 猪价、能繁母猪存栏 |
| 地产 | 销售面积同比、土地溢价率 |

**判读规则**：景气指标处于历史 30 分位以下 + PB 接近历史底部 → 双低信号。

#### 5. EV/EBITDA 作为辅助估值

| 指标 | 优势 | 适用 |
|---|---|---|
| EV/EBITDA | 剔除资本结构与折旧影响 | 重资产周期股（钢铁、化工、地产） |
| EV/产能 | 按吨产能/万吨估值 | 大宗商品（吨钢市值、吨煤市值） |
| EV/储量 | 按可采储量估值 | 资源股（油气、矿产） |

这些"实物量估值"在周期底部尤其有用——业绩为负时，资产价值仍可衡量。

---

### 四、调整后的周期股估值矩阵

| 指标 | 普通股 | **强周期股** |
|---|---|---|
| 主估值锚 | TTM PE | **PB** |
| EPS 输入 | TTM EPS | **5–7 年平均 EPS** |
| 历史分位 | PE 分位 | **PB 分位** |
| PEG | 使用 | **不使用**（增速波动太大） |
| 行业对比 | 行业 PE 中位数 | **行业 PB 中位数 + 景气位置** |
| 辅助指标 | — | **EV/EBITDA + 实物量估值** |
| 外部锚 | 无 | **行业景气度指标** |

---

### 五、周期股版本的三角验证

```
                       V₁ ── 历史 PB 中位数 × 当前净资产
                        \
当前市值 ─── 偏离度 ───  估值锚区间  ← V₂ ── 平均 EPS × 周期 PE 中枢
                        /
                       V₃ ── 行业 EV/EBITDA 中位数 × 当前 EBITDA

附加判断：景气度指标当前分位 → 决定取区间的上沿/中位/下沿
```

**景气低位时取 V_low 作为底部支撑，景气高位时取 V_high 作为顶部预警**。

---

### 六、周期股核心操作原则

| 原则 | 说明 |
|---|---|
| **逆向思维** | PE 高时（业绩差）布局，PE 低时（业绩好）减仓 |
| **不用 PEG** | 周期股增速波动剧烈，PEG 完全无意义 |
| **盯紧景气** | 商品价格、库存、产能利用率比财务报表更领先 |
| **警惕"价值陷阱"** | 看似低 PE 的周期顶部 = 最大的陷阱 |
| **左侧布局** | 等右侧业绩反转时，股价通常已涨 30%+ |
| **永续假设失效** | DCF 模型在周期股上慎用，永续增长率概念不适用 |

# valuation.py

# ─── 通用 ───
def is_cyclical(stock) -> bool:
    """判断是否为强周期股（行业 + 波动率双重检验）"""

# ─── 普通股 PE 路径 ───
def pe_matrix(stock) -> dict:
    """六维度扫描，返回各项信号 + 汇总正面数"""

def pe_triangulation(stock) -> dict:
    """PEG / 历史分位 / 行业相对 三方法计算目标市值，输出锚区间"""

def pe_scenario(stock) -> dict:
    """三档情景 → 悲观/中性/乐观估值区间"""

# ─── 周期股 PB 路径 ───
def normalized_eps(stock, years=7) -> float:
    """穿越周期 EPS：取 N 年平均"""

def pb_percentile(stock) -> float:
    """PB 历史分位"""

def cycle_position(stock) -> str:
    """结合 PB 分位 + 行业景气度，输出周期位置：底部/复苏/顶部/衰退"""

def cyclical_valuation(stock) -> dict:
    """周期股专属估值流程"""

# ─── 主入口 ───
def valuation_workflow(stock):
    if is_cyclical(stock):
        return cyclical_valuation(stock)
    return pe_matrix(stock) | pe_triangulation(stock) | pe_scenario(stock)

## 关键提醒

1. **PE 不是真理，是参考**——任何单一指标都会失效，必须组合使用
2. **不预测目标价，只测算区间**——所有输出以区间形式呈现
3. **周期股反着看**——传统估值逻辑在周期股上完全反向
4. **估值是入场券，不是判决书**——估值通过后还要走"验证排雷"模块
5. **行业差异 > 个股差异**——先判断行业属性，再选估值方法
