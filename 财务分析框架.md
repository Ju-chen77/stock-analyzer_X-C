# 财务分析框架

> 三段式分析：**业绩检验 → 业绩归因 → 验证排雷**
> 对应一份报告的三个章节：**是什么 / 为什么 / 有没有雷**

---

## 一、业绩检验

**核心问题**：报表数字过关吗？盈利是否换得来真金白银？

### Part A：常规盈利指标

| 指标 | 说明 |
|---|---|
| 营收增速（YoY / QoQ） | 成长性的起点 |
| 归母净利润增速 | 与营收增速对比看效率 |
| 毛利率、净利率 | 盈利能力 |
| ROE | 综合回报水平 |

**图表**：✅ 5 年时间序列折线图（增速 + 利润率叠加）

### Part B：利润表 vs 现金流量表 三组对照

**核心逻辑**：利润表看"应计"，现金流量表看"实收实付"。两者持续背离，说明业绩有水分。

| 对比组 | 利润表科目 | 现金流量表科目 | 健康信号 |
|---|---|---|---|
| **收入含金量** | 营业收入 | 销售商品、提供劳务收到的现金 | 现金流入 / 营收 ≈ 1.0–1.17（含增值税） |
| **成本含金量** | 营业成本 | 购买商品、接受劳务支付的现金 | 比值稳定，无大幅背离 |
| **利润含金量** | 净利润 | 经营活动现金流量净额 | 经营现金流 / 净利润 ≥ 0.8 |

**判读规则**：
- 收入对比：现金流入持续低于营收 → 应收账款堆积，可能是赊销冲量
- 成本对比：现金流出远高于成本 → 大量备货或预付，可能是行业景气下滑前兆
- 利润对比：净利润持续高于经营现金流 → 利润"纸面化"，是排雷重点信号

**图表**：✅ 三组双柱状图，每组并排显示利润表项 vs 现金流量表项，辅以"比值"折线叠加在第二坐标轴。

```
┌─ 收入 vs 销售收到现金 ──┐  ┌─ 成本 vs 购买支付现金 ──┐  ┌─ 净利润 vs 经营现金流 ─┐
│   柱+比值线              │  │   柱+比值线              │  │   柱+比值线             │
└──────────────────────────┘  └──────────────────────────┘  └─────────────────────────┘
```

5 年时间序列，一眼看出趋势是收敛还是发散。

## 二、业绩归因

**核心问题**：业绩好/差，源头在哪？增长（或下滑）靠什么驱动？

| 归因层级 | 指标 |
|---|---|
| ROE 杜邦分解 | 净利率 × 总资产周转 × 权益乘数 |
| 营收拆解 | 量（销量）× 价（ASP），或分业务 / 分地区 |
| 毛利率归因 | 原材料价格、产品结构、规模效应 |
| 费用变动 | 销售、管理、研发、财务费用率 |

**图表**：✅ 杜邦分解瀑布图 + 营收结构堆叠柱状图（按业务线或区域）

## 三、验证排雷

**核心问题**：数字背后有没有雷？

| 雷区 | 指标 | 警戒信号 |
|---|---|---|
| 收入质量 | 应收账款增速 vs 营收增速 | 应收增速持续高于营收 |
| 库存压力 | 存货增速 vs 营收增速 | 存货大幅领先营收 |
| 利润质量 | 经营现金流 / 净利润 | 持续 < 0.8 或为负 |
| 商誉风险 | 商誉 / 净资产 | > 30% |
| 偿债风险 | 资产负债率、利息保障倍数 | 趋势恶化（看行业） |
| 治理风险 | 大股东质押比例、审计意见 | 质押 > 50%、非标准意见 |

**图表**：✅ "排雷仪表盘"——红黄绿三色标注每个指标的当前状态

## 代码模块映射

```
financials.py     → 业绩检验（指标计算 + 三组现金流对照）
attribution.py    → 业绩归因（杜邦分解、营收拆解）
risk_check.py     → 验证排雷（红线检测、仪表盘输出）
```

### 关键函数

```python
# financials.py
def revenue_cash_match(stmt)        # 收入 vs 销售收到现金
def cost_cash_match(stmt)           # 成本 vs 购买支付现金
def profit_cash_match(stmt)         # 净利润 vs 经营现金流净额
def plot_three_match_charts(...)    # 三组对照图表

# attribution.py
def dupont_decompose(stmt)          # ROE 三因子分解
def revenue_breakdown(stmt)         # 营收按业务/地区拆解

# risk_check.py
def receivables_check(stmt)         # 应收 vs 营收
def inventory_check(stmt)           # 存货 vs 营收
def goodwill_check(stmt)            # 商誉占比
def risk_dashboard(stmt)            # 红黄绿仪表盘汇总
```

## 设计原则

1. **三段式不可跳过**：业绩好不代表没雷，业绩差也要找原因——三块互补，缺一不可
2. **图表先于文字**：每段分析都要有可视化产出，便于快速判读
3. **行业差异化**：金融、地产、周期股的警戒线需单独设定，不可套用通用阈值
4. **横向 + 纵向**：每个指标既要看自身 5 年趋势，也要看行业分位数
