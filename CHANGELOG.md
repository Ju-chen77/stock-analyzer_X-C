## [app_V4.7] — 2026-06-14

### 重构

- **Tab ① 业绩检验按「业绩检验框架（补充）」三层重组**：Part A 盈利能力 / Part B 业绩自洽性 / Part C 现金流验证 / Part D 主营构成 / Part E·F 资产负债堆积，在原有版本基础上增补内容

### 新增

- **A 层盈利能力增补**：
  - KPI 卡片增加「扣非净利率」「ROIC」（共 7 项）
  - 利润率趋势图增补「营业利润率」「扣非净利率」（毛利率/营业利润率/净利率/扣非净利率四线）
  - 新增「回报率趋势」图：ROE / ROA / ROIC（排除杠杆看真实回报）
  - 新增「增长质量判读」：营收增速 vs 扣非净利润增速，识别低基数效应与非经常损益拉动
- **B 层业绩自洽性**（新增章节）：营业收入 vs 合同负债双柱+YoY 图（未来订单领先指标）；应收/存货自洽性仍见 Tab ③
- **C 层现金流验证增补**：
  - 新增「自由现金流 FCF」图（经营现金流 − 资本开支），红绿区分正负
  - 新增「三类现金流组合」图（经营/投资/筹资）+ 逐年 +/− 组合标签条，映射 8 种公司阶段（健康成熟/成长扩张/靠融资生存/变卖资产等）
- **扣非净利润数据源**：新增 `_fetch_deduct_indicator()`，额外调用 `stock_financial_analysis_indicator`（新浪源）补齐扣非净利润，失败容错不阻塞主流程

### 说明

- 行业对比（毛利率 vs 同行中位数、行业分位雷达图）需循环抓取同行数据，有阻断风险，留待后续单独接入

## [app_V4.5] — 2026-06-13

### 重构

- **Tab ③ 验证排雷按「验证排雷框架」重组**：从 5 项扁平指标升级为 A 资产端 / B 偿债杠杆 两大分组维度（共 10 项），全部基于已抓取的三大报表结构化数据

### 新增

- **一票否决总判定卡片**（D-2）：任一红灯 → 不通过（估值作废）；≥3 黄灯 → 警示（打折观察）；无红灯且黄灯≤2 → 通过；全绿 → 优质。顶部醒目展示 🔴🟡🟢 计数
- **红黄绿历史热力图**（D-3）：10 项指标 × 近 6 年的状态矩阵，颜色随年份漂移直观呈现风险演变趋势（"看演变，不看快照"）
- **资产端 5 项**：商誉占比、收入质量（应收−营收增速）、库存压力（存货−营收增速）、其他应收占比、研发资本化（Δ开发支出口径）
- **偿债杠杆 5 项**：资产负债率、**有息负债/货币资金（康美/康得新造假红线）**、速动比率、利息保障倍数、现金流偿债覆盖
- **行业差异化阈值提示**（E）：说明银行/地产/证券/周期股需专门规则，治理维度（质押/审计/监管）需额外数据源待接入

### 修复

- 研发资本化比例口径修正：从「累计开发支出余额 / 研发费用」（存量÷流量，导致低研发企业如茅台误报红灯）改为「Δ开发支出 /（研发费用+Δ开发支出）」（流量÷流量），茅台从误报"不通过"修正为"通过"

## [app_V4.4] — 2026-06-13

### 重构

- **Tab ② 业绩归因四层递进重组**：按「业绩归因框架」重新组织为第一层 ROE 杜邦分解、第三层利润率拆解、第四层现金流验证三大模块

### 新增

- **杜邦贡献度量化卡片**（第一层）：计算 ROE 同比变化中净利率、周转率、杠杆各贡献多少 pct，标注主导因子

- **现金流验证总结**（第四层）：三组验证卡片（收入真实性、利润真实性、FCF）+ 通过/关注/不通过 判定，归因结论必须通过验证才能采信

### 改进

- `compute_attribution()` 扩展：新增毛利率、营业利润率、扣非净利率计算；新增瀑布图数据生成；新增现金流三组验证逻辑

## [app_V4.1] — 2026-06-12

- **Render 云端部署**：新增 `render.yaml` + `requirements.txt`，后端部署至 https://stock-analyzer-x-c.onrender.com
- **CORS 跨域支持**：后端 `app_V4.py` 新增 `Access-Control-Allow-Origin: *` 响应头，允许前端从任意域名调用 API
- **公网可用**：任何人可通过浏览器访问，无需本地运行 Python

### 改进

- **Part B 现金流对照图**：柱状图改为双折线图，去掉比值（ratio）右轴，更直观对比两组数据走势
- **东方财富直接 API**：用 `push2.eastmoney.com` 替代有 bug 的 `stock_individual_info_em`，稳定获取行业名称和总股本
- **PE/PB 口径统一**：历史 PE 和当前 PE 均使用年报 EPS 计算，避免东财"动态 PE"（季度年化）导致百分位失真
- **一致预期年份修正**：优先匹配当前年份（2026）的预测列，而非第一个匹配列

### 修复

- `stock_individual_info_em` pandas 报错（ValueError: Length mismatch）→ 改用东方财富直接 HTTP API
- 一致预期取到 2025 年旧数据 → 优先匹配 2026 列
- PE 百分位 = 0%（动态 PE 与年报 PE 口径不一致）→ 统一用年报 EPS 基准
- `fetch_price()` 备用路径也用 `_get_spot()` 替代已废弃的 `stock_individual_info_em`

## [app_V4] — 2026-06-12

### 新增

- **三段式财务分析框架**（对应 `财务分析框架.md`）
  - **Tab ① 业绩检验**
    - KPI 卡片：最新一期营收 / 净利润 / 毛利率 / 净利率 / ROE
    - 营收 & 净利润柱状图 + YoY 增速折线（双轴）
    - 毛利率 / 净利率 / ROE 趋势折线
    - 三组现金流对照图（收入含金量 / 成本含金量 / 利润含金量）
  - **Tab ② 业绩归因**
    - 杜邦三因子分解图（净利率 × 总资产周转 × 权益乘数 = ROE）
    - 杜邦数字明细表（5 年历史）
    - 销售 / 管理 / 研发 / 财务 费用率趋势折线
  - **Tab ③ 验证排雷**
    - 五项排雷仪表盘（红黄绿三色）：利润含金量 / 收入质量 / 库存压力 / 商誉风险 / 偿债风险
    - 应收账款增速 vs 营收增速趋势图
    - 存货增速 vs 营收增速趋势图
  - **Tab ④ 估值区间**（对应 `PE_PB估值法（akshare）.md`）
    - PE 六维矩阵信号卡（TTM PE / 历史分位 / Forward PE / PEG / PB 分位 / EPS 增速）
    - 历史 PE-TTM 折线图（近 5 年 + 中位数线 + 当前值线）
    - 历史 PB 折线图（近 5 年 + 中位数线 + 当前值线）
    - 三角验证目标价区间（低 / 中 / 高）
    - 三档情景分析表（悲观 / 中性 / 乐观）

- 前端改用 **Plotly.js** 交互式图表（CDN 加载），支持悬浮查看数值
- 后端新增 `_get_equity()`：兼容 6 种新浪资产负债表权益列名写法，找不到时用「资产 − 负债」反推
- 后端新增 `_get_pe_history()`：用月线价格 × 年报 EPS/BVPS 本地计算历史 PE/PB 序列
- 后端新增 `compute_valuation()`：PE/PB 矩阵 + 三角验证 + 情景分析完整估值流程

### 修复

- **日期格式 bug**：`_set_date()` 加 `.replace("-", "")` 兼容 `"2016-03-31"` 带横线格式，修复年报过滤全部失败、指标显示 N/A 的问题（由合作者 Claude 检出）
- **权益乘数 / ROE 不显示**：扩展权益列名匹配范围，修复杜邦分解三因子缺失的问题
- **PE/PB 折线图不显示**：显式声明 `xaxis.type:'date'`，去掉 `fill:'tozeroy'` + 透明底色组合（该组合会把线条压到填充层之下）
- **Tab 切换图表贴底**：改为懒渲染，切到对应 Tab 时才调用 Plotly.newPlot()，修复隐藏容器高度为 0 导致所有数据点压在 y=0 的问题
- `fcol()` 改为精确匹配优先，再做子串匹配，减少列名误命中
