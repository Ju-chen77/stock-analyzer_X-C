## [app_V5.3] — 2026-06-18

### 修复

- **图表横向滚动条**：业绩检验（Tab①）资产/负债结构两列图表会把页面撑出横向滚动条。
  根因——`render()` 中 `renderTab('t1')` 在 `outEl` 仍为 `display:none` 时执行，Plotly 读到容器宽度 0、回退到默认 ~700px；叠加 CSS Grid 的 `1fr`（即 `minmax(auto,1fr)`）以内容固有宽度作为列下限，两列被撑出视口。
  修复：① `render()` 先显示 `outEl` 再渲染 Tab①；② `.charts-2col` / `.charts-3col` 改用 `minmax(0,1fr)`、`.chart-wrap` 加 `min-width:0` 作为兜底——任何标签页都不会再撑出横向滚动条

### 调整（配色 · 马卡龙色系）

- **堆积面积图（资产 / 负债结构）**：高饱和色板改为低饱和**马卡龙色系**（玫瑰粉 `#F2A6B3` / 薄荷绿 `#A8DEC0` / 天空蓝 `#A9C7EC` / 薰衣草紫 `#C9B6E4` / 蜜桃橙 `#F7C6A3` / 水鸭青 `#9BD6D2` / 奶油黄 `#F5E1A4`）；去掉色块间分界线（`line.width=0`），色块无缝相接
- **热力图**：敏感性热力图（Tab⑥）与红黄绿状态图（Tab③）统一为马卡龙 红 `#F49096` → 黄 `#F8D777` → 绿 `#8FD3A6`（较堆积图略提饱和）；状态图保留灰 `#e2e8f0` 表示无数据

## [app_V5.1] — 2026-06-18

### 新增

- **盈利预测层（Tab ⑤ 盈利预测）**：按「盈利预测框架」新增预测模块，位于估值之前的前置环节，永远输出区间与情景、假设全部显式化。新增 `compute_forecast()`，复用主分析已抓取的历史利润表，无额外网络请求
  - **关键假设面板**：营收增速（3/5年CAGR加权）、毛利率、期间费用率（含规模效应摊薄）、有效税率、归母净利率锚，全部显式列出便于复盘
  - **三档情景**（悲观/基准/乐观）：营收→毛利→费用→税→归母净利润→EPS，并衔接历史中位 PE 给出对应目标价
  - **基准 3 年轨迹**：复利外推 T+1/T+2/T+3 营收与归母净利润
  - **敏感性矩阵**：营收增速 × 毛利率 5×5 热力图 → 归母净利润
  - **单变量冲击**：营收增速/毛利率/费用率/有效税率单独变动对归母净利润的影响（直观印证"毛利率假设错 2pct 比营收错 10% 影响更大"）
- 框架文档入库：`盈利预测框架.md`（含落地说明，记录可行性调整）

### 可行性调整

- **方法 C（量价模型）已从框架删除**：分产品销量/ASP/行业规模/市占率三种数据源（新浪/东财 emweb/akshare 其余接口）均无法提供，无法落地，已从 `盈利预测框架.md` 移除；收入预测以方法 A + 方法 B 为准
- **方法 B（一致预期）/ 预期差量化暂未实现**：`stock_profit_forecast_em` 等一致预期接口在本地代理环境不可用。本模块以方法 A（历史 CAGR）+ 历史假设落地，方法 B 待可直连环境补充
- **净利润推导改用「归母净利率锚」**：以当前实际归母净利率为基准，按毛利率/费用率相对基准的偏离（税后）调整，而非纯自下而上 营收×毛利率−费用−税——后者对投资收益/减值/少数股东损益占比大的控股型公司（如 TCL，归母 45 亿但经营口径仅 ~16 亿）会系统性失真

### 调整

- Tab 顺序：盈利预测置于 ④估值区间 之后、量价印证之前，显示为 ⑤；量价印证顺延为 ⑥（内部 id 保持 t5/t6 不变）
