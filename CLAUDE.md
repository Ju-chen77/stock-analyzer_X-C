# Stock Analyzer X-C — Project Framework

## 项目简介

基于 Python 的 **A 股基本面与量价综合分析平台**。通过 **akshare** 接入企业财务数据与市场数据，构建涵盖 **业绩检验 → 业绩归因 → 验证排雷 → 行业对比 → 盈利预测 → 估值 → 量价印证** 的完整分析体系，输出企业的**合理估值区间**与**成长空间**评估。

---

## 核心理念

> **本项目不预测股价，而是评估"估值区间"与"成长空间"。**

- 所有预测以**情景分析**形式呈现（悲观 / 中性 / 乐观）
- 估值输出**区间**而非具体目标价
- 所有关键假设必须**显式化**并做敏感性分析
- 用"当前市值 vs 内在价值区间"的**偏离度**，由使用者自行判断
- 红灯一票否决：排雷阶段不允许用其他绿灯抵消单一红灯

## 分析架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        akshare 数据层 + 缓存                          │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────────┐
│                       基本面分析三段式                                │
│                                                                      │
│   业绩检验  ───→  业绩归因  ───→  验证排雷                          │
│   是什么          为什么          有没有雷                            │
│                                                                      │
│   ↑               ↑               ↑                                  │
│   └───────────────┴───────────────┘                                  │
│           行业对比（横向支撑）                                         │
│           A 精选同行 + B 行业分位                                     │
└─────────────────────────────────────────────────────────────────────┘
                                  ↓ 排雷通过
┌─────────────────────────────────────────────────────────────────────┐
│                       盈利预测 → 估值                                 │
│                                                                      │
│   三档情景 EPS  ───→  PE/PB 估值法  ───→  估值区间 + 增长空间        │
└─────────────────────────────────────────────────────────────────────┘
                                  ↕ 横切印证
┌─────────────────────────────────────────────────────────────────────┐
│                         量价数据印证层                                │
│   场景 A 业绩窗口 │ B 排雷增强 │ C 估值修复 │ D 行业轮动              │
└─────────────────────────────────────────────────────────────────────┘
```

**核心原则**：基本面优先，量价印证；先检验再归因，归因后排雷；排雷通过才估值；估值输出区间不输出目标价。

## 前置条件

启动前必须确认：

1. Python 3.10+ 环境
2. 已安装 akshare：`pip install akshare --upgrade`
3. 已配置 `.env`（如需 Claude API 用于文本分析）
4. 了解 akshare 的现状：
   - 公开数据源，**无显式调用配额但需 sleep 防封 IP**
   - 接口可能因源站改版变动，需定期 `pip install --upgrade`
   - 一致预期数据对小盘股 / ST 股**覆盖度有限**
   - EBITDA、分产品销量等数据**akshare 不直接提供**

## 开发阶段

### 阶段一：数据基础设施（地基）

- [ ] `scraper.py`：封装 akshare 常用接口，加 sleep + 重试
- [ ] `cache.py`：两层缓存（L1 行业池 90 天 / L2 个股结果 7 天）
- [ ] `statements.py`：报表清洗，中文字段映射到英文 key
- [ ] `industry_mapping.py`：东财 ↔ 申万映射表（基础工程，阻塞所有行业对比功能）
- [ ] `main.py`：跑通"输入股票代码 → 抓取 → 清洗 → 落盘"

### 阶段二：财务分析三段式

- [ ] `financials.py`：业绩检验
  - Part A 盈利能力指标
  - Part B 业绩自洽性（4 组对照）
  - Part C 现金流验证（三组含金量 + FCF + 三类组合）
- [ ] `attribution.py`：业绩归因
  - ROE 杜邦分解
  - 营收拆解
  - 利润率瀑布
  - 现金流验证（与 financials.py 共用）
- [ ] `risk_check.py`：验证排雷
  - A 资产端 / B 偿债 / C 治理三大维度
  - D 红黄绿仪表盘 + 一票否决
  - E 行业差异化阈值

### 阶段三：行业对比

- [ ] `industry_compare.py`
  - A 路径：8 维相似度筛选 → Top 15 + 龙头锚 + 双基准
  - B 路径：各指标行业分位 + 雷达 + 综合定位标签
  - A 和 B 共享行业核心指标数据底座

### 阶段四：盈利预测与估值

- [ ] `forecast.py`：盈利预测
  - 收入三方法（CAGR / 一致预期 / 量价模型）
  - 毛利率三层逻辑
  - 三档情景（悲观/基准/乐观）
  - 敏感性分析 + 预期差量化
- [ ] `valuation.py`：PE/PB 估值
  - PE 六维矩阵 + 三角验证
  - 周期股 PB 路径（PB 替代 PE、穿越周期 EPS、景气度叠加）

### 阶段五：量价数据印证

- [ ] `market_signal.py`
  - 场景 A 业绩披露窗口期监测
  - 场景 B 排雷信号增强
  - 场景 C 估值修复确认
  - 场景 D 资金流向 / 行业轮动
  - 基本面 × 量价四象限定位

### 阶段六：网页发布

- [ ] Web UI 整合
- [ ] 可视化交互（仪表盘、雷达图、热力图、四象限）
- [ ] 报告导出（PDF / Markdown）
- [ ] （可选）年报 PDF 解析（pdfplumber + Claude API）

## 关键设计原则

### 估值原则

1. **估值不是预测，估值输出区间**：禁止单一目标价
2. **三档情景必备**：悲观 / 基准 / 乐观，任何输出都要带情景标签
3. **关键假设必须显式化**：每个数字背后都要可追溯到假设和理由
4. **敏感性分析优先于精确度**：与其纠结小数点，不如把范围算清楚

### 数据原则

5. **缓存优先**：用户查询路径上**绝不允许有 akshare 实时调用**
6. **批量接口优先**：`stock_zh_a_spot_em()` 等批量接口减少调用次数
7. **共享数据底座**：行业对比 A/B 共用一份数据，避免重复抓取
8. **多源交叉**：报表 + 公告 + 审计意见 + 市场信号，单一来源不可靠

### 方法论原则

9. **三段式不可跳过**：业绩好不代表没雷，业绩差也要找原因
10. **红灯一票否决**：排雷阶段不允许加权平均掩盖单点风险
11. **趋势 > 绝对值**：所有指标看 3-5 年趋势，单期数据不算数
12. **行业差异化**：金融、地产、周期股需专门规则，通用阈值仅作初筛
13. **A 股 CAS 准则**：所有指标公式以 CAS 为准，不直接套 US GAAP

### 工程原则

14. **不确定的地方写 `# TODO:`**，不要乱猜
15. **API 抓取必须加错误处理**（try/except + 重试 + 配额检查）
16. **API key 存在 `.env`**，绝对不能硬编码

## 编码规范

- 每个函数必须写注释，说明：**输入参数、返回值、功能描述、所用会计假设**
- 不确定的地方写 `# TODO: 说明`，不要乱猜
- 数据抓取必须加 try/except + 重试机制 + sleep 防封 IP
- 估值函数必须**显式返回假设参数**，便于复盘
- 配置项（阈值、TTL 等）写在 `settings.py`，不要硬编码

### 函数注释模板

```python
def calculate_dcf(
    fcf_series: list[float],
    wacc: float,
    terminal_growth: float
) -> dict:
    """
    基于自由现金流折现法计算企业内在价值
    
    Args:
        fcf_series: 未来 5–10 年预测自由现金流（单位：亿元）
        wacc: 加权平均资本成本（小数，如 0.09 表示 9%）
        terminal_growth: 永续增长率（建议 2%–3%）
    
    Returns:
        dict: {
            "intrinsic_value": 内在价值（亿元）,
            "assumptions": 输入假设字典,
            "sensitivity": 对 wacc 与 g 的敏感性矩阵
        }
    
    会计假设:
        FCF = 经营性现金流 - 资本性支出（CAS 口径）
    """
```
